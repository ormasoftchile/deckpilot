## Key Findings

### Rendering Pipeline (Full Stack)

1. **DeckParser** ‚Üí parses YAML frontmatter (gray-matter), delegates to SlideParser
2. **SlideParser** ‚Üí ordered pipeline:
   - Checkpoint extraction
   - Voice-over stripping
   - Action block parsing (replaced with placeholders)
   - **Layout directive transformation** (`transformLayoutDirectives()`) ‚Äî string-based preprocessing
   - **markdown-it rendering** ‚Üí HTML
   - Placeholder injection ‚Üí action links
   - Fragment processing ‚Üí `<span class="fragment">`
3. **WebviewProvider** ‚Üí serializes all slides into `window.deckData` JSON
4. **Image URLs** ‚Üí transformed at send-time (not parse-time)

### Layout Directive System

- **Directives:** `:::center`, `:::columns`/`:::left`/`:::right`, `:::advanced`, `:::optional`, auto-close
- **Implementation:** String-based preprocessing (not markdown-it plugin) ‚Äî replaces lines before renderer
- **CSS classes:** `.layout-center`, `.layout-columns` (1fr 1fr hardcoded), `.layout-left`, `.layout-right`, `.disclosure-advanced`, `.step-optional`

### Identified Pain Points

1. **Column ratios hardcoded** ‚Äî no support for asymmetric splits (e.g., 2:1 code-heavy layouts)
2. **No gap/alignment configurability** ‚Äî all CSS is fixed
3. **Disclosure labels hardcoded English** ‚Äî not localizable or author-configurable
4. **Onboarding mode breaks centering** ‚Äî `align-items: flex-start` overrides `:::center` blocks
5. **No advanced layout variants** ‚Äî no `:::split`, `:::grid`, 3-col, sidebar options
6. **Scroll risk inside columns** ‚Äî `#slide-content` max-width may cause unwanted scrolling

### Extractability Insight

**The rendering pipeline is pure TypeScript with zero VS Code dependencies:**
- `transformLayoutDirectives()`, `parseSlides()`, `injectBlockElementsFromParsed()` all callable in headless Node.js
- Slide HTML can be rendered to string and inspected in Mocha tests
- `WebviewProvider` is VS Code-entangled (only the delivery layer)

## Decisions Proposed (for team inbox)

1. **Column ratios:** Extend syntax to `:::columns 2:1` and map to grid template values
2. **Integration testing:** Add layout-rendering pipeline tests without VS Code
3. **Disclosure labels:** Allow `:::advanced Custom Label` for author override
4. **Onboarding centering:** Decide if `:::center` should override onboarding's left-alignment
