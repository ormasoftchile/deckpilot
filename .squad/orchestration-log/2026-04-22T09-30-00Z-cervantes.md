### Architectural Analysis

**Key Finding:** The feature is entirely parser-layer scoped. Conductor and Webview layers are completely unchanged.

```
 ┌────────────────┐   ┌────────────────┐
 │  .deck.md      │   │  .deck.yaml    │
 │  (inline mode)  │   │  (sidecar)     │
 └───────┬────────┘   └───────┬────────┘
         │                     │
         ▼                     ▼
   DeckParser.parseDeck()   SidecarLoader.load()
         │                     │
         └────────┬────────────┘
                  ▼
           MergeEngine.merge()  ← NEW
                  │
                  ▼
           Deck (canonical model)  ← EXISTING, extended
                  │
                  ▼
            Conductor  ← UNCHANGED
```

**Model Changes:**
- `Slide` gains `id?: string` field
- `DeckMetadata` gains optional `recording`, `export`, `environment` sections
- `Slide` gains optional `cues`, `duration`, `timingHint` fields
- `ActionDefinition` requires no changes (already supports all needed action types)

**Merge Precedence:** inline frontmatter > sidecar YAML > global defaults (simple shallow-merge per slide)

### Risk Assessment

| Risk | Severity | Mitigation |
|------|----------|-----------|
| Slide ID stability | Medium | Use HTML comments or frontmatter `id:` field; auto-generate as fallback with warnings |
| Sidecar file watcher complexity | Low | Mirror existing `.deck.env` watcher pattern |
| **Scope creep (Phase 3)** | **HIGH** | **DEFER entirely** — "reusable metadata templates" and "multi-file imports" deferred. Phase 1+2 ships all user value. |
| Existing decks must not break | Critical | Sidecar path optional at every level; field is optional, auto-generated IDs don't change behavior |
| Validation diagnostics | Medium | Ship incrementally; don't block Phase 1 on all 7+ rules |

## Decision Points Settled

✅ **Phase 3 killed.** Templates and multi-file imports are windmills — solutions looking for problems we don't have yet.

✅ **Merge precedence simple.** Inline-wins rule is clear and predictable. No cascading or deep inheritance.

✅ **Sidecar always optional.** If `.deck.yaml` is absent or malformed, `parseDeck()` behaves identically. Existing decks unaffected.

✅ **Parser layer only.** No Conductor or Webview changes needed. Minimal architectural surface.
